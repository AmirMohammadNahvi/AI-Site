# Goal

Operate as FaralYar's UX and interaction quality system.

This skill focuses on the quality of the experience, not only how things look.

# Scope

Use this skill for:
- usability improvements
- interaction patterns
- state design
- onboarding quality
- trust and reassurance
- friction reduction
- conversion-oriented UX
- empty, loading, success, and error state quality
- form behavior
- keyboard and mobile ergonomics
- perceived product quality

# Core principle

A great FaralYar UX should make the user feel:
- guided
- confident
- in control
- not overloaded
- rewarded for progressing
- safe when paying or changing data
- impressed by smoothness, not distracted by gimmicks

# UX rules

Always evaluate:
- does the user know what to do next?
- does the system respond clearly?
- does the page feel lighter than the task difficulty?
- are loading and waiting moments handled well?
- is the user reassured during payment/auth/risky actions?
- is success celebrated clearly but briefly?
- is failure recoverable?

# Chat UX rules

For chat surfaces:
- first-use experience must be welcoming
- empty state must guide action
- composer must feel responsive
- upload behavior must feel integrated
- model switching must not feel risky or confusing
- long sessions must remain easy to scan

# Auth UX rules

For signup, login, and OTP:
- reduce perceived effort
- explain what happens next
- make retry and recovery obvious
- avoid intimidating validation language

# Pricing and billing UX rules

For pricing and checkout:
- reduce purchase hesitation
- explain value before complexity
- make quota and plan differences understandable
- reassure users during payment and verification

# Dashboard and admin UX rules

For dashboard and admin:
- keep high-density surfaces understandable
- reduce accidental misuse
- support fast scanning
- support confident action-taking

# Interaction quality rules

Always define if relevant:
- hover
- focus
- active
- disabled
- loading
- success
- error
- empty
- interrupted
- resumed

# Conversion principle

A premium UX increases willingness to subscribe when:
- the product feels reliable
- users understand value quickly
- the product reduces friction at key decisions
- users feel progress, not confusion
- waiting and risk moments are handled gracefully
