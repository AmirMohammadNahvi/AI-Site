# Admin patterns

## Lists
- strong page title
- filters near the data
- obvious create action
- readable statuses
- dense but calm rows

## Forms
- group fields by purpose
- separate dangerous fields
- separate public-facing text from technical config
- keep helper text precise

## Destructive actions
- isolate visually
- require clear confirmation
- explain impact

## Financial and sensitive settings
- add stronger confirmation and helper text
- reduce accidental edits
- expose status and last-known state clearly
