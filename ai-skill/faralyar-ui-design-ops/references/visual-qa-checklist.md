# Visual QA checklist

## Typography
- Persian wrapping is natural
- headings are clearly distinguished
- placeholder text is readable
- body text contrast is acceptable
- helper text does not disappear

## Layout
- section rhythm is consistent
- cards do not all have the same weight
- no giant dead areas without purpose
- shell hierarchy is readable
- dense layouts remain scannable

## Controls
- buttons, inputs, selects, and dropdowns align in height
- focus state is visible in both themes
- disabled states are distinct
- destructive actions are clearly separated
- upload controls feel custom and intentional

## Dark mode
- surfaces are layered
- borders are visible enough
- background is not oppressively dark
- accents are restrained
- panels do not melt into background
- chat shell, sidebar, composer, and cards are clearly separated

## Light mode
- cards do not disappear into page
- white areas do not feel empty
- body text keeps enough contrast
- tinted backgrounds remain subtle
- hierarchy does not flatten

## Mobile
- no horizontal scroll
- composer remains usable with keyboard open
- modal content is not clipped
- tap targets remain comfortable
- fixed elements do not collide
- drawer and sidebar behavior is clear
- attachment rows wrap safely

## RTL
- icon direction is correct
- chevrons and back patterns respect rtl
- chip and badge spacing feels intentional
- mixed Persian and English content wraps acceptably
- alignment logic feels native, not mirrored lazily

## Empty, loading, and error states
- empty states are useful
- loading states do not cause layout jump
- error states are readable and actionable
- pending and success states are visually distinct

## Route-specific reminders
### Chat
- upload must never look native
- model picker must not dominate small screens
- empty state should help the user start

### Pricing
- recommended plan must be obvious
- comparison must remain readable on mobile

### Settings and actions
- destructive actions must be visually isolated
- modal and sheet overflow must be safe

### Dashboard and admin
- dense tables and forms must remain calm and readable
- create/edit/list state differences must be obvious
